Setup¶

Setting up Uveb is done in 8 steps. Currently, Uveb is not ready for a stable release. Please keep in mind the security issues.

Requirements¶

Uveb requires the following prerequisites:

Name Ubuntu Untested Arch CentOS Untested
python3 apt install python3 pacman -S python yum install centos-release-scl && yum install rh-python36
pip apt install python3-pip pacman -S python-pip yum install python36-setuptools && easy_install-3.6 pip
virtualenv pip3 install virtualenv pacman -S python-virtualenv pip3 install virtualenv
mariadb pacman -S mariadb-server pacman -S mariadb yum install mariadb-server

Clone¶

Clone Uveb through git

git clone https://github.com/UWC-CSC/uveb.git && cd uveb

Virtualenv¶

I recommend installing a virtualenv, however if you do not wish to use virtualenv you are free to skip this step altogether, however incorrect setup may occur.

virtualenv venv
. venv/bin/activate

Dependencies¶

Install the required modules. Please read what requirements.txt entails before you run the following command.

pip install -r requirements.txt

MySQL¶

Start the MySQL server:

systemctl start mysqld

Optionally, you may choose to enable the MySQL server on startup:

systemctl enable mysqld

Database Setup¶

The database you installed is empy and you must set it up before proceeding.

Login as the root user:

mysql -u root -p

Create the uveb database which will contain all of the data of the application:

CREATE DATABASE uveb;

Create the user which will use the database:

GRANT ALL PRIVILEGES ON *.* TO 'uveb_username'@'localhost' IDENTIFIED BY 'secure_password';
FLUSH PRIVILEGES;

Dump all of the tables from the provided schema.sql (read it before you execute the command!):

mysql -u uveb_username --password=secure_password uveb < uveb/schema.sql

Uveb Config¶

Uveb itself is relatively easy to configure. Please check the configuration manual for more information. Uveb must be manually configured.

Run¶

Now that everything is configured, you may run the application.

export FLASK_APP=uveb
export FLASK_ENV=development
flask run --host=0.0.0.0
